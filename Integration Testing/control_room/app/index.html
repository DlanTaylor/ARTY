The Project
Design

The Automated Rescue Transportation Unit (ARTY), a chest sized device, is designed for a quick field response of helpless victims in need of an auxiliary rescue. ARTY is also designed to be portable and ideal for pool related activities (i.e. public pools & waterparks). When dispatched, ARTY is steered towards the drowning victim from a radio-controlled transmitter or using wireless technology. Moreover, using text-to-speech technology, ARTY notifies close individuals of a possible drowning. This device is also designed to sustain victims up to 125 lbs., and shaped as a catamaran watercraft, in order to optimize buoyancy and device control.

Circuit Schematic

For the circuit, we have a dual control system that runs with two Arduino Nanos. One Nano is responsible for RC control. The module is connected to a RC receiver. When the signal is received from the RC controller, the microprocessor will send signals to the output pins to control the motor drivers. The other Nano is controlled by a Raspberry Pi through serial communication. When the Pi receives commands from user interface, it will send signals to the Nano and move ARTY accordingly as well. The main challenge of the circuit is that both Nanos outputs are connected to same input pins of the motor drivers. The microprocessors are running the risk of shorting out each other in this case. Thus, we used 3 IC7486, or XOR logic gates, for the input pins of the microcontrollers and output signals to the motor drivers. The XOR gate returns true when the inputs are odd and false when both inputs are high. This will ensure the safety of the microprocessors and motor drivers.

Programming

Following is the prgramming tools we used for this project: Node.js, Python, C, Nginx, Shell Script, Bootstrap, HTML, Javascript, CSS, and Git. The Node.js runs majority of backend tasks, which includes control the motor drivers and get the sensor values. The Python is reponsible for the AI calculation and stream it to a network port. Nginx is a server we use on the Pi to manage the user requests. The Arduino Nanos are running in C. The Shell Script is used for glue everything together on the Pi when some of the programs do not like to cooperate (ie. running speaker on server, get some of the Python program to work with Node.js). Git is used for version control. And of course, this site is running with Bootstrap that allows the users to view it on multiple platform, which includes mobile devices we originally target for.

Testing

For our pool test, we ran the ARTY unit multiple times. The unit moved forward, reverse, right, and left successfully. We added weights (15, 40, and 50 LBS) to the unit, and ARTY was able to carry the weights for couple rounds as well.
